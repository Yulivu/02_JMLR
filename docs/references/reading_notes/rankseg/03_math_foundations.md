# 数学基础慢读：概率、Dice 和 Bayes 规则

这份笔记专门补阅读 RankSEG 需要的数学背景。它不是严格数学教材，而是为了让你看论文公式时不至于被符号挡住。

## 1. 向量和下标

论文里经常写：

```text
X in R^d
Y in {0, 1}^d
```

意思是：

```text
X = (X_1, X_2, ..., X_d)
Y = (Y_1, Y_2, ..., Y_d)
```

`d` 是像素或 feature 的数量。

如果是一张很小的 `2 x 3` 图片，那么可以把 6 个像素排成一列：

```text
d = 6
```

真实标签也就是 6 个 0/1：

```text
Y = [1, 1, 0, 0, 1, 0]
```

这表示第 1、2、5 个像素属于目标区域。

## 2. 0/1 向量和集合是同一回事

论文经常在向量和集合之间切换。

如果：

```text
Y = [1, 1, 0, 0, 1, 0]
```

那么：

```text
I(Y) = {1, 2, 5}
```

`I(Y)` 就是真实为 1 的位置集合。

如果模型预测：

```text
delta(X) = [1, 0, 0, 0, 1, 1]
```

那么：

```text
I(delta(X)) = {1, 5, 6}
```

交集是：

```text
I(Y) ∩ I(delta(X)) = {1, 5}
```

表示预测对的前景像素。

## 3. 向量内积 Y^T delta(X)

论文里写：

```text
Y^T delta(X)
```

这就是两个向量对应位置相乘再求和：

```text
Y^T delta(X) = sum_j Y_j * delta_j(X)
```

因为 `Y_j` 和 `delta_j(X)` 都是 0/1，所以只有两者都等于 1 时才贡献 1。

因此：

```text
Y^T delta(X) = TP
```

也就是 true positive 的数量。

例如：

```text
Y        = [1, 1, 0, 0, 1, 0]
delta(X)= [1, 0, 0, 0, 1, 1]
```

逐项相乘：

```text
[1, 0, 0, 0, 1, 0]
```

求和得到：

```text
2
```

这表示预测对了 2 个前景像素。

## 4. L1 范数 ||Y||_1

论文写：

```text
||Y||_1
```

对于 0/1 向量，它就是里面有多少个 1。

例如：

```text
Y = [1, 1, 0, 0, 1, 0]
||Y||_1 = 3
```

所以：

```text
||Y||_1 = 真实前景像素数量
||delta(X)||_1 = 预测前景像素数量
```

## 5. Dice 公式慢拆

论文的 Dice 写成：

```text
Dice_gamma(delta)
= E[(2Y^T delta(X) + gamma) / (||Y||_1 + ||delta(X)||_1 + gamma)]
```

先忽略最外面的 `E` 和 `gamma`，核心是：

```text
2Y^T delta(X) / (||Y||_1 + ||delta(X)||_1)
```

翻译成：

```text
2 * 预测对的前景数量 / (真实前景数量 + 预测前景数量)
```

为什么分子乘 2？因为真实区域和预测区域的交集同时属于真实区域和预测区域，Dice 会给交集双倍权重。

## 6. 期望 E 是什么

论文不是只评价一张图片，而是评价从数据分布中抽到的所有可能图片的平均表现。因此写：

```text
E(...)
```

表示期望，也可以先粗略理解成“长期平均”。

如果验证集有很多图片，我们会算：

```text
每张图的 Dice
然后取平均
```

理论里的 `E` 就是这个平均的概率版本。

## 7. 条件概率 p_j(x)

论文最重要的概率是：

```text
p_j(x) = P(Y_j = 1 | X = x)
```

读法：

```text
在已经看到输入 x 的情况下，第 j 个像素真实为 1 的概率。
```

注意，它不是模型输出本身，而是真实数据分布中的理想概率。

现实里我们不知道 `p_j(x)`，只能训练模型估计它：

```text
q_hat_j(x) ≈ p_j(x)
```

## 8. Bernoulli 随机变量

Bernoulli 随机变量就是只会取 0 或 1 的随机变量。

如果：

```text
B_j(x) ~ Bernoulli(p_j(x))
```

意思是：

```text
B_j(x) = 1 的概率是 p_j(x)
B_j(x) = 0 的概率是 1 - p_j(x)
```

在论文里，`B_j(x)` 可以理解成“第 `j` 个像素真实标签的随机版本”。

## 9. Poisson-binomial 分布

这是论文里一个名字很吓人的概念，但意思可以很朴素。

假设有很多个独立的 0/1 随机变量：

```text
B_1, B_2, ..., B_d
```

它们成功概率不一定一样：

```text
B_1 成功概率 p_1
B_2 成功概率 p_2
...
B_d 成功概率 p_d
```

现在把它们加起来：

```text
Gamma = B_1 + B_2 + ... + B_d
```

那么 `Gamma` 表示“一共有多少个成功”。这个 `Gamma` 服从 Poisson-binomial 分布。

如果所有成功概率都一样，例如都等于 `p`，那就是普通 binomial 分布。

如果每个成功概率都可以不一样，那就是 Poisson-binomial 分布。

在分割里：

```text
Gamma(x) = 真实前景像素数量的随机版本
```

它的均值是：

```text
mu = sum_j p_j
```

这很直观：每个像素贡献一个“成为前景的概率”，加起来就是预期前景大小。

它的方差是：

```text
sigma^2 = sum_j p_j(1 - p_j)
```

如果很多像素概率接近 0 或 1，不确定性小，方差也小；如果很多像素概率接近 0.5，不确定性大，方差会更大。

## 10. Bayes rule 是什么

“Bayes rule”在统计学习里通常表示理论最优决策规则。

在分类里，如果目标是最小化 0/1 分类错误，那么 Bayes 分类器是：

```text
如果 P(Y = 1 | X = x) >= 0.5，预测 1
否则预测 0
```

这是普通二分类下的理论最优。

但 RankSEG 论文说：对于 Dice 分割，Bayes rule 不是简单的 `0.5` 阈值。

Dice 分割的 Bayes rule 是：

```text
把 p_j(x) 排序，选择前 tau*(x) 个。
```

所以，不同评价指标会有不同 Bayes rule。

这是整篇论文的关键思想。

## 11. Calibration 是什么

论文里有一个词：

```text
Dice-calibrated
```

可以先理解成“一致性”。

如果一个方法在理论上拥有无限数据、无限模型能力时，能恢复 Dice 指标下的 Bayes 最优规则，那么它就是 Dice-calibrated。

普通 cross-entropy 对分类是 calibrated 的，因为它最终能学到真实条件概率，然后用 0.5 阈值能得到分类 Bayes rule。

但论文证明：

```text
cross-entropy + 固定 0.5 阈值
```

对 Dice 分割不是 Dice-calibrated。

原因不是 cross-entropy 不能估计概率，而是后面的固定阈值决策不符合 Dice 的 Bayes rule。

## 12. Excess risk 是什么

论文 Theorem 11 讨论：

```text
Dice(delta*) - Dice(delta_hat)
```

这里：

```text
delta* = 理论最优 Bayes 分割
delta_hat = 实际 RankDice 分割
```

因为 `delta*` 是最优，所以：

```text
Dice(delta*) >= Dice(delta_hat)
```

两者差距就是 excess risk。它表示：

```text
实际方法距离理论最优还差多少。
```

Theorem 11 说，这个差距可以被概率估计误差控制：

```text
Dice(delta*) - Dice(delta_hat)
<= C * E ||q_hat(X) - p(X)||_1
```

翻译成人话：

```text
只要模型估计的概率 q_hat 接近真实概率 p，
RankDice 的 Dice 分数就接近理论最优。
```

这就是 RankDice 理论部分最核心的保证。

## 13. 读公式时的建议

看到一个复杂公式时，可以按四步读：

1. 先找它在优化什么，例如 Dice、IoU、tau。
2. 再找每个符号的含义，例如 `p_j`、`J_tau`、`Gamma`。
3. 再看它是不是在求和，求和通常是在枚举“所有可能情况”。
4. 最后才看证明细节，不要一开始就被分母和上下标拖住。

对于这篇论文，最值得掌握的是：

```text
Dice 是整体指标；
固定阈值是逐像素决策；
RankDice 用排序 + 体积选择把两者对齐。
```
