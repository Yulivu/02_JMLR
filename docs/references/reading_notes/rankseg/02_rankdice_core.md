# 第 2 节：RankDice 核心方法

这一节是全文最重要的部分。读懂这里，就基本读懂了 RankSEG 的主线。

## 1. 先看作者的基本假设

论文首先考虑二分类分割，也就是每个像素只有 0/1 两种标签。

作者定义：

```text
p_j(x) = P(Y_j = 1 | X = x)
```

这表示：给定输入图像 `x` 后，第 `j` 个像素真实属于目标区域的概率。

论文还假设：

```text
Y_i 独立于 Y_j，条件是 X 已知
```

写成公式是：

```text
Y_i ⟂ Y_j | X,  i != j
```

这句话不要理解成“图片里的像素真的毫无关系”。它是一个概率建模假设：当整张输入 `X` 已经给定后，每个像素标签的不确定性可以用各自的条件概率描述。论文附录 B.3 还解释了，常用 cross-entropy 损失其实也隐含类似假设。

作者还假设不同像素的 `p_j(x)` 基本不会完全相等。这个假设主要是为了避免排序时出现并列，让理论表述更干净。

## 2. Theorem 1 在说什么

Theorem 1 给出 Dice 指标下的 Bayes 最优分割规则。

先把“Bayes 最优”翻译成人话：

```text
如果我知道真实的条件概率 p_j(x)，
那么在所有可能的预测规则里，
哪一种能让期望 Dice 最大？
```

Theorem 1 的答案是：

```text
先把 p_j(x) 从大到小排序，
然后选择前 tau*(x) 个像素。
```

也就是说，Bayes 最优预测不是“所有概率超过 0.5 的像素”，而是：

```text
概率排名最高的前 tau*(x) 个像素
```

其中 `tau*(x)` 是这张图的最优预测体积，也就是应该选多少个像素。

## 3. 公式里的 tau 是什么

`tau` 可以理解成预测出来的前景像素数量：

```text
tau = ||delta(x)||_1
```

如果 `tau = 0`，就是一个像素都不选。

如果 `tau = 100`，就是选出概率最高的 100 个像素。

如果 `tau = d`，就是把全部像素都预测成前景。

Theorem 1 的关键是：对每一个可能的 `tau = 0, 1, ..., d`，都可以计算一个期望 Dice 分数，然后选择分数最大的那个 `tau`。

## 4. J_tau(x) 是什么

论文里有：

```text
J_tau(x)
```

它表示概率最高的前 `tau` 个像素的下标集合。

例如一共 5 个像素，条件概率是：

```text
p(x) = [0.20, 0.91, 0.77, 0.11, 0.62]
```

从大到小排序是：

```text
第 2 个像素：0.91
第 3 个像素：0.77
第 5 个像素：0.62
第 1 个像素：0.20
第 4 个像素：0.11
```

如果 `tau = 3`，那么：

```text
J_3(x) = {2, 3, 5}
```

也就是选第 2、3、5 个像素。

## 5. Gamma 和 Gamma_minus_j 是什么

Theorem 1 里最吓人的部分是：

```text
Gamma(x)
Gamma_minus_j(x)
```

它们其实是在数“真实标签里一共有多少个 1”。

定义：

```text
Gamma(x) = sum_{j=1}^d B_j(x)
```

其中：

```text
B_j(x) ~ Bernoulli(p_j(x))
```

可以理解为：第 `j` 个像素真实是不是前景，是一个 0/1 随机变量，成功概率是 `p_j(x)`。

所以：

```text
Gamma(x)
```

表示整张图真实前景像素数量的随机版本。

而：

```text
Gamma_minus_j(x)
```

表示把第 `j` 个像素去掉之后，其他像素中真实为 1 的数量。

如果 `Gamma_minus_j(x) = l`，意思是：除了第 `j` 个像素以外，还有 `l` 个真实前景像素。

## 6. 为什么公式里有 tau + l + gamma + 1

Dice 的分母是：

```text
真实前景数量 + 预测前景数量 + gamma
```

预测前景数量就是 `tau`。

当我们讨论某个被选中的像素 `j` 对 Dice 分子的贡献时，只有 `Y_j = 1` 时它才贡献 true positive。此时真实前景数量可以拆成：

```text
Y_j + 其他真实前景数量 = 1 + l
```

所以分母变成：

```text
tau + l + 1 + gamma
```

这就是公式里：

```text
tau + l + gamma + 1
```

的来源。

分子里的 `2` 来自 Dice 的 `2TP`。

于是某个被选中像素 `j` 的期望贡献大概长这样：

```text
2 * p_j(x) * P(Gamma_minus_j(x) = l) / (tau + l + gamma + 1)
```

这句话可以分成三层理解：

1. `p_j(x)`：第 `j` 个像素真的为 1 的概率。
2. `P(Gamma_minus_j(x) = l)`：其他像素里有 `l` 个真实为 1 的概率。
3. 分母：如果发生这种情况，Dice 的分母是多少。

然后对所有可能的 `l` 求和，就得到这个像素在预测体积为 `tau` 时的期望贡献。

## 7. Theorem 1 的两阶段结构

Theorem 1 最重要的不是复杂公式，而是它把 Bayes 最优分割拆成了两件事：

第一件事：排序。

```text
谁的 p_j(x) 大，谁就更应该被选。
```

第二件事：选数量。

```text
到底选前多少个，才让期望 Dice 最大？
```

这个结构非常关键。因为它告诉我们，Dice 最优预测不需要在 `2^d` 个 0/1 向量里暴力搜索。原本每个像素都可以选或不选，有 `2^d` 种组合；但 Theorem 1 说，只需要考虑：

```text
tau = 0, 1, ..., d
```

也就是 `d + 1` 种预测体积。

这就是 RankDice 能实际做起来的理论基础。

## 8. 为什么固定阈值是次优的

如果使用固定阈值，比如 0.5，那么每张图都遵循：

```text
p_j(x) >= 0.5 就选
p_j(x) < 0.5 就不选
```

但 Theorem 1 表明，最优规则是：

```text
选概率最高的前 tau*(x) 个
```

这等价于一个阈值：

```text
threshold(x) = 第 tau*(x) 大的概率
```

注意这个阈值依赖于 `x`。不同图片的最佳阈值可能完全不同。

所以固定阈值的问题是：

```text
它强迫所有图片使用同一个截断线，
而 Dice 最优规则要求每张图片有自己的截断线。
```

这就是 Remark 2 的意思。

## 9. RankDice 三步法

RankDice 是 Theorem 1 的 plug-in 版本。

所谓 plug-in，就是：

```text
理论公式里需要真实 p_j(x)，
现实里不知道 p_j(x)，
那就先估计一个 q_hat_j(x)，
再把 q_hat 塞进理论公式里。
```

### 第一步：条件概率估计

论文用 cross-entropy 来估计：

```text
q_hat(x)
```

其中 `q_hat_j(x)` 是模型估计的：

```text
P(Y_j = 1 | X = x)
```

这一步和普通分割训练很像。模型可以是 CNN，也可以是 U-Net、DeepLab、PSPNet 等。

### 第二步：排序

对新输入 `x`，把估计概率从大到小排：

```text
q_hat_{j_1}(x) >= q_hat_{j_2}(x) >= ... >= q_hat_{j_d}(x)
```

这里 `j_1` 是概率最大的像素下标，`j_2` 是第二大的像素下标。

### 第三步：估计体积

对每个可能的 `tau`，计算一个估计 Dice 分数：

```text
pi_tau(x)
```

然后取最大：

```text
tau_hat(x) = argmax_tau pi_tau(x)
```

最后预测：

```text
选 q_hat 最高的前 tau_hat(x) 个像素
```

## 10. 一个小例子

假设一张图只有 6 个像素，模型估计概率为：

```text
[0.83, 0.72, 0.49, 0.48, 0.31, 0.06]
```

固定阈值 0.5 会选择前两个：

```text
[1, 1, 0, 0, 0, 0]
```

RankDice 会先排序，然后比较不同体积：

```text
tau = 0：一个都不选
tau = 1：选 0.83
tau = 2：选 0.83, 0.72
tau = 3：选 0.83, 0.72, 0.49
tau = 4：选 0.83, 0.72, 0.49, 0.48
...
```

如果考虑整张图的期望 Dice 后，发现 `tau = 4` 最好，那么 RankDice 会选 4 个，即使其中两个概率低于 0.5。

这就是它和固定阈值的根本区别。

## 11. 读这一节时最应该记住什么

Theorem 1 告诉我们：

```text
Dice 最优预测 = 概率排序 + 最优体积选择
```

RankDice 告诉我们：

```text
真实概率不知道，就先用模型估计概率；
再排序；
再估计最优体积；
最后选前 tau 个。
```

所以这篇论文的核心创新不是发明一个新神经网络，而是改变“概率到分割 mask”的最后一步决策规则。
