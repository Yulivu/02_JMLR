# 第 1 节：背景、问题和动机

这一节对应论文 Introduction、Existing methods 和 Our contribution。它的任务不是推公式，而是告诉读者：为什么已有分割框架在 Dice/IoU 目标下可能不是最优。

## 1. 分割任务在做什么

论文把一个输入记作：

```text
X in R^d
```

可以把 `X` 想成一张图片，也可以想成一段文本或其他有很多 feature 的对象。`d` 表示一共有多少个像素或特征。例如一张 `512 x 512` 的图片，如果只看一个通道，像素数就是 `d = 512 * 512`。

真实标签记作：

```text
Y in {0, 1}^d
```

也就是说，`Y` 是一个长度为 `d` 的 0/1 向量。第 `j` 个位置：

```text
Y_j = 1
```

表示第 `j` 个像素属于目标区域；如果：

```text
Y_j = 0
```

表示它不属于目标区域。

模型给出的预测分割函数记作：

```text
delta: R^d -> {0, 1}^d
```

输入一张图 `X`，输出一个 0/1 向量 `delta(X)`。其中：

```text
delta_j(X) = 1
```

表示模型认为第 `j` 个像素是前景或目标区域。

论文还用集合记法：

```text
I(Y) = {j: Y_j = 1}
I(delta(X)) = {j: delta_j(X) = 1}
```

这只是把 0/1 向量换成“哪些位置被选中了”的集合。

## 2. Dice 和 IoU 是什么

Dice 和 IoU 都是在衡量预测区域和真实区域的重合程度。

如果用集合直觉：

```text
真实区域 = I(Y)
预测区域 = I(delta(X))
```

那么：

```text
交集 = 预测对的区域
并集 = 真实区域和预测区域合起来
```

Dice 的核心形式是：

```text
Dice = 2 * |真实区域 ∩ 预测区域| / (|真实区域| + |预测区域|)
```

IoU 的核心形式是：

```text
IoU = |真实区域 ∩ 预测区域| / |真实区域 ∪ 预测区域|
```

如果用机器学习常见的 TP、FP、FN 来看：

```text
Dice = 2TP / (2TP + FP + FN)
IoU  = TP / (TP + FP + FN)
```

其中：

```text
TP = 预测为 1 且真实为 1 的像素数
FP = 预测为 1 但真实为 0 的像素数
FN = 预测为 0 但真实为 1 的像素数
```

论文的公式里还有一个 `gamma >= 0`，叫 smoothing parameter。它的作用是让分母不要太容易出现极端情况，尤其是空集合时。直觉上可以先把 `gamma` 看成一个稳定项。

## 3. 为什么 Dice/IoU 和普通分类不一样

普通二分类常常关心每个样本单独对不对。比如判断一个像素是不是前景，可以对每个像素做：

```text
如果 P(Y_j = 1 | X) >= 0.5，就预测 1
否则预测 0
```

这在“每个像素单独分类正确率”这个目标下很自然。

但 Dice/IoU 不只是逐像素准确率。它们是整张图上的比例：

```text
分子：预测区域和真实区域的重合
分母：真实区域、预测区域、错误区域的组合
```

这意味着某个像素要不要选，不只取决于它自己的概率，也取决于你已经选了多少像素、整张图真实目标大概有多大、其他像素概率分布是什么样。

这就是论文的核心矛盾：

```text
训练时常用逐像素分类损失，
预测时常用固定阈值，
但评估时用的是整张图的 Dice/IoU。
```

目标不一致，就可能导致理论最优解和实际框架输出不一致。

## 4. 传统固定阈值框架

论文把传统框架写成两步：

第一步，学一个概率函数：

```text
q_hat = argmin_q empirical_loss(q) + regularization
```

这里 `q_j(x)` 表示模型认为第 `j` 个像素为 1 的概率。

第二步，用固定阈值预测：

```text
delta_hat(x) = 1(q_hat(x) >= 0.5)
```

也就是说，每个像素只看自己的概率是否超过 0.5。

论文讨论了两类常见 loss。

第一类是 classification-based losses，例如：

```text
cross-entropy
focal loss
```

它们本质上来自分类问题。

第二类是 Dice-approximating losses，例如：

```text
soft-Dice
Lovasz-softmax
Tversky loss
```

它们试图直接近似 Dice 或 IoU，但论文认为这些方法也不一定能保证得到真正 Dice 最优的分割规则。

## 5. 固定阈值为什么会失败：最简单例子

论文在 Lemma 9 的证明里给了一个非常适合理解的例子。

假设只有两个像素：

```text
d = 2
p_1 = P(Y_1 = 1 | X) = 0.45
p_2 = P(Y_2 = 1 | X) = 0.44
```

如果用固定阈值 0.5：

```text
p_1 < 0.5
p_2 < 0.5
```

所以预测是：

```text
(0, 0)
```

也就是一个像素都不选。

但论文证明，在 Dice 目标下，Bayes 最优规则会选择：

```text
(1, 1)
```

为什么？直觉是：虽然两个概率都没超过 0.5，但它们都接近 0.5，而且如果目标区域确实存在，什么都不选会让 Dice 很差。Dice 更在意预测区域和真实区域的整体重合，而不是单个像素是否超过 0.5。

这个例子说明：

```text
“概率低于 0.5”不等于“在 Dice 最优预测中一定不该选”。
```

## 6. 本文提出的替代方案

论文提出 RankSEG。针对 Dice，叫 RankDice；针对 IoU，叫 RankIoU。

RankDice 的核心是：

1. 估计每个像素为前景的概率。
2. 按概率从大到小排序。
3. 不固定阈值，而是估计应该选前多少个像素。
4. 选出概率最高的前 `tau` 个像素。

所以它的预测阈值不是固定的 0.5，而是由当前输入 `x` 决定。

假设某张图里最优选择数量是 `tau = 1200`，那么阈值就是第 1200 大概率对应的那个概率。另一张图里 `tau` 可能是 300 或 8000，阈值也随之变化。

这就是论文所说的 adaptive thresholding。

## 7. 论文贡献怎么写

论文在 Introduction 里列了几个贡献，可以整理成四句话：

1. 提出 RankDice，这是一个针对 Dice 指标一致的 ranking-based segmentation framework。
2. 给出 Dice 分割的 Bayes 最优规则，并定义 Dice-calibrated。
3. 证明传统固定阈值框架配合很多分类损失不是 Dice-calibrated，而 RankDice 是 Dice-calibrated。
4. 为高维分割设计了可扩展计算算法，并在模拟和真实数据集上验证效果。

如果之后要模仿这篇 JMLR 写作，最值得学习的是它的结构：

```text
先指出传统框架和评估指标不一致
再给 Bayes 最优规则
再提出 plug-in 方法
再证明 calibration 和 excess risk
最后用实验验证
```
