Project 2 · Tensorized Regular-Language Posterior Algebra for CRFs

CRF 正则语言后验事件概率与训练信号

目标是让序列模型不只是通过 hard constraint 输出一个合法答案，而是在后验分布中真正把概率质量放到合法结构上。 我们计算并训练 CRF 后验中落入规则语言 \(L\) 的总概率 \(P_\theta(L\mid x)\)，把业务规则从解码后的硬修正变成模型内部可审计、可训练的概率信号。

Hard constraint 修最后答案；我们训练模型的后验信念。
要算什么 \(P_\theta(L\mid x)\)：模型有多少概率相信规则集合 \(L\)
怎么计算 CRF 概率 × DFA 规则监控 → 合法路径总和
怎么训练 用 \(-\log P_\theta(L\mid x)\) 推动后验质量进入合法结构
当前证据 公式 sanity + 梯度机制 + 受控/半真实/真实来源小字段 probe

核心直觉：问题不是“能不能修成合法”，而是“模型原本有多相信合法”

假设 OCR 模型看到一个编号字段，肉眼像 AB123。业务规则要求编号必须是： 两个大写字母 + 三个数字。最高分输出 AB123 可能是合法的，但模型内部还会给很多候选编号分配概率。

AB1230.30合法
AB1280.10合法
AC1230.05合法
A81230.20非法
ABB230.12非法
AB12C0.08非法

我们要问的新问题

不是“最高分答案是不是合法”，而是：

\[ \begin{aligned} P_\theta(L_{\mathrm{CODE}}\mid x) &= 0.30 + 0.10 + 0.05 \\ &= 0.45. \end{aligned} \]

这表示模型只有 45% 的后验概率支持合法编号格式。Hard constraint 可以把最后答案修成合法， 但它看不到剩下 55% 的概率质量还在非法候选上。

Hard constraint 修最后答案；\(P_\theta(L\mid x)\) 衡量并训练模型对合法结构的后验信念。

先把几个词翻译成人话

CRF 一种给“整条标签序列”分配分数和概率的模型。它不是只保留一个答案，而是维护很多候选答案的概率分布。
正则语言 \(L\) 一组可以用规则描述的合法序列。例如编号必须是“两位字母 + 三位数字”。
\(P_\theta(L\mid x)\) 给定输入 \(x\) 后，所有满足规则 \(L\) 的候选序列概率加起来。直观上就是模型对规则的相信程度。
后验概率质量 把一整类候选答案的概率相加，而不是只看最高分答案。
DFA monitor 一个规则监控器。每读入一个标签，就更新“目前是否还符合规则”。
Semi-event training 少标注时，只用“答案应该满足某条规则”这个事件训练模型，不要求每条样本都有完整标签。

为什么这不是 regex checker，也不只是 constrained decoding

这不是 regex checker。regex checker 问：这条输出是否合法？
这也不只是 constrained decoding。constrained decoding 问：能不能强制输出合法？
本项目问：模型的整个后验分布，有多少概率质量相信合法结构？这个概率能否被计算、被训练、被误差分析？

输出层

传统硬约束主要管最终答案：最后交付给用户的序列必须合法。

后验层

我们关心模型内部概率：合法答案集合在整个后验里占多大比例。

训练层

如果 \(P_\theta(L\mid x)\) 可微，就能作为训练信号，让模型少把概率放在非法结构上。

理论层

这个量有精确定义、动态规划计算、资源上界和近似误差边界，不只是工程后处理。

\(P_\theta(L\mid x)\) 的三个价值

可靠性审计 发现“最高分输出合法，但模型大部分概率还在非法答案上”的隐藏冲突样本。
弱监督训练 在少标注或未标注输入上，用规则事件 \(L\) 给模型提供结构信号。
规则与学习结合 规则不再只在解码后修答案，而是进入模型后验分布和训练目标。
误差可讨论 如果近似计算 \(P_\theta(L\mid x)\)，可以追踪事件概率误差，而不是只报告最终输出。

当前可以稳妥宣称什么

  • \(P_\theta(L\mid x)\) 可计算、可审计、可训练。
  • product automaton transfer 与 brute-force sanity 在小规模对象上一致。
  • 受控、半真实、真实来源小字段 probes 支持结构信号正向。
  • 弱监督/少标注格式事件是当前最有潜力的定位。

当前证据摘要：机制支持，正式 benchmark 还没有

半真实字段验证说明了什么 这里的任务模拟真实表单/OCR 字段：金额、日期、剂量、商品编号都有明确格式规则。 “semi-event 训练”指的是：除了少量有标注样本，还给模型一些未标注输入，只告诉它输出应该满足格式规则 \(L\)。 如果训练后 \(P_\theta(L\mid x)\) 上升，意思是模型后验里更多概率质量落到了合法格式上；如果字符准确率或完整匹配准确率也上升，说明这种结构信号没有只是在“刷规则”，还可能帮助实际识别。

证据如何读

证据块能支持什么不能支持什么

梯度机制检查\(-\log P_\theta(L\mid x)\) 会压低非法转移、抬高部分合法转移不等于性能一定提升
受控/半真实/真实来源小字段结构信号方向正向，部分任务指标正向不等于 benchmark superiority
\(\lambda\) sweep规则项强度有 tradeoff，需要平衡监督目标不能说规则项越大越好

和常见方法相比，我们回答的问题不同

方法做什么改变原始后验吗能否用于训练边界
Hard constraint decoding 修最终输出，让答案满足规则。 通常不改变原始后验。 通常不能直接训练模型的后验信念。 可能隐藏模型内部冲突。
Rule feature 把规则作为局部或全局特征加入模型。 可以影响模型。 可以。 依赖特征设计；不一定直接得到事件概率。
Posterior regularization 对后验分布施加约束或期望条件。 可以。 可以。 框架更泛，是必须保留的 baseline。
本项目 直接计算并训练正则语言事件概率 \(P_\theta(L\mid x)\)。 可以通过训练项改变原始后验。 可以，尤其适合少标注/弱监督格式事件。 仍需 faithful baseline 和 benchmark 审计。
这里不宣称“我们全面更好”。更准确的定位是：hard constraint、rule feature、posterior regularization 和本项目回答的问题不同；正式实验必须把它们作为强对照。

适用场景：不是抽象数学玩具

OCR 编号识别 发票号、商品码、订单号通常有固定格式。我们可以问模型有多少后验概率相信这个格式。
表单字段抽取 日期、金额、剂量等字段有明显规则。规则事件可以作为少标注训练信号。
Slot filling 字段顺序、必填/禁填组合可以写成事件 \(L\)，用来诊断模型是否遵守结构。
NER / BIO P6 前的 canonical benchmark：即使 constrained decoding 输出合法，也要检查后验里是否仍有大量非法 BIO 概率质量。

idea 来源：uMPS 给的启发，以及我们真正转向的地方

核心灵感来自 AISTATS 2021 的 uMPS 论文 Tensor Networks for Probabilistic Sequence Modeling。 它让我们看到一件关键事情：正则语言可以成为概率模型内部的可计算事件。

uMPS 里做什么我们转向什么
给字符串 \(s\) 建模，计算 \(P(s)\)给输入 \(x\) 后的标签序列 \(y\) 建模，计算 \(P_\theta(y\mid x)\)
问 \(P(s\in R)\)：生成字符串是否落在 regex 事件里问 \(P_\theta(y\in L\mid x)\)：CRF 后验是否相信合法结构
重点是生成式序列概率模型重点是结构化预测、后验查询、训练信号和误差分析
本项目不是“把 uMPS 改成 CRF”。我们借用的是“规则可以是概率事件”这个视角，然后把中心对象固定为 \(Z_{\theta,L}(x)\) 和 \(P_\theta(L\mid x)\)。

系统架构：输入、规则、后验概率怎么连起来

前面解释的是为什么有用；这里解释计算流程。可以把系统看成一条流水线：

输入 \(x\)文本、OCR 字符、编号字段或序列特征。
CRF 打分给每条候选序列 \(y\) 一个权重。
DFA 监控同步判断路径是否仍符合规则 \(L\)。
合法路径求和只累计最终被 DFA 接受的路径。
输出概率\(P_\theta(L\mid x)=Z_{\theta,L}(x)/Z_\theta(x)\)。
\[ Z_\theta(x)=\sum_y \exp(\mathrm{score}_\theta(x,y)), \qquad Z_{\theta,L}(x)=\sum_{y\in L}\exp(\mathrm{score}_\theta(x,y)). \] \[ P_\theta(L\mid x)=\frac{Z_{\theta,L}(x)}{Z_\theta(x)}. \]

计算上，我们把 CRF 的 context \(c\) 和 DFA 的 state \(q\) 合成 product state \((c,q)\)， 每一步同时做局部打分、CRF 状态更新和规则状态更新。

\[ M_t^L[(c,q),(c',q')] = \sum_j G_t(c,j)\, \mathbf{1}\{c'=U(c,j)\}\, \mathbf{1}\{q'=\delta(q,j)\}. \] \[ Z_{\theta,L}(x)=u_0^\top M_1^L M_2^L\cdots M_T^L b_L. \]

理论可信度：为什么这不是纯工程 trick

这一节想说明什么

前面解释的是为什么有用；下面解释为什么它是一个可被形式化、可被误差分析的研究对象。

精确恒等式

有限 CRF event posterior identity 和 product automaton transfer identity 已通过 statement-level proof audit。

索引边界

T=1、first-order reduced、order-\(m\) context 的双计分风险已修复并通过二轮 audit。

Appendix support

条件 MPO/rank membership 只作为附录支持，不作为主贡献；主文聚焦 posterior event semantics。

误差传播

在 positive-cone domination 条件下，可以把 transfer 近似误差传到 \(Z_{\theta,L}(x)\) 事件质量边界。

\[ (1-\rho_t)M_t^L \le \widehat M_t^L \le (1+\rho_t)M_t^L \quad\Longrightarrow\quad \widehat Z_{\theta,L}(x)\ \text{有乘法型误差边界}. \]
这里的理论主张是有条件的：不说任意 CRF / DFA 都低秩；不把 rank/MPO 当成 paper identity；不把 rank sanity 当成实证 usefulness。

Scope boundaries：边界必须写清楚

当前可以稳妥说

  • \(P_\theta(L\mid x)\) 是清楚的新中心对象。
  • 它可计算、可审计、可作为训练信号。
  • local probes 支持结构信号正向，尤其是少标注/弱监督格式事件。
  • 理论链条已经通过多轮 proof audit，可继续正式化。

仍然不能说

  • 不能说 JMLR-ready 或完整投稿就绪。
  • 不能说 benchmark superiority。
  • 不能说全面优于 hard constraint / WFST / posterior regularization。
  • 不能说任意 CRF / DFA 都低秩。

下一步最该补什么

  1. 补一个 canonical BIO/NER public slice，展示 decoded legality 不等于 posterior consistency。
  2. 保留 retail_fields_v1 作为辅助 real-source small-field，而不是主 benchmark。
  3. 继续保留 hard constraint / rule feature / posterior regularization 作为强 baseline，并报告 hidden conflict cases。
  4. 把 rank/MPO 放到 appendix，主文集中讲 posterior event semantics、event training 和 diagnostic。

一句话收束

我们把规则从输出后的过滤器，变成模型后验中的可计算概率事件，再进一步变成少标注场景下可训练的结构信号。
