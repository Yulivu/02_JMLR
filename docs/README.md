# Documentation Index

Updated: 2026-05-29

This directory is organized by purpose. Top-level `docs/` should stay small.

## Start Here

- `HANDOFF_FOR_PAPER_WRITING.md`: main handoff for a supervisor or paper writer.
- `EXPERIMENT_INVENTORY.md`: experiment block, config, script, result, and claim map.
- `PROJECT_OVERVIEW.md`: current project state, safe claims, and reading order.

## Manuscript Preparation

- `manuscript/FINAL_CLAIM_TABLE.md`: supported and forbidden claims.
- `manuscript/PAPER_POSITIONING.md`: paper identity and reviewer-facing framing.
- `manuscript/JMLR_METHODS_OUTLINE.md`: methods-paper outline.
- `manuscript/THEORY_PROOF_PROSE.md`: current formal proof prose.
- `manuscript/RELATED_WORK_DRAFT.md`: related-work draft and citation spine.
- `manuscript/REPRODUCIBILITY_PACKAGE_CHECKLIST.md`: reproducibility status.

## External Review

- `external_review/EXTERNAL_REVIEW_README.md`: shortest entrypoint for an external reviewer.
- `external_review/EXTERNAL_REVIEW_PACKET_CURRENT.md`: current one-file review packet with latest evidence and uncertainty-baseline result.

## Protocols

- `protocols/EXPERIMENT_PLAN.md`: formal run plan and experiment definitions.
- `protocols/BIO_NER_SLICE_PROTOCOL.md`: WNUT17 BIO/NER slice protocol.
- `protocols/R5_WNUT17_FORMAL_PROTOCOL.md`: R5 two-regime protocol and results boundary.
- `protocols/B7_WFST_DESIGN_NOTE.md`: WFST-style baseline decision note.
- `protocols/DIAGNOSTIC_REANALYSIS_PLAN.md`: diagnostic reanalysis plan and metrics.

## Evidence

- `evidence/EVIDENCE_AND_AUDIT.md`: evidence and claim audit.
- `evidence/THEORY_AUDIT.md`: theory audit, proof consistency, and guardrails.

## Runbooks

- `runbooks/AUTODL_HPC_RUNBOOK.md`: project-specific AutoDL runbook.
- `runbooks/GITHUB_AUTODL_FILEZILLA_WORKFLOW.md`: general GitHub/AutoDL/FileZilla workflow note.

## References

- `references/REFERENCE_INDEX.md`: reference and reading-note index.
- `references/papers/`: small tracked reference PDFs.
- `references/reading_notes/`: retained reading notes.

## Cleanup Policy

Early idea packets, obsolete route checklists, duplicate outline drafts, stale presentation HTML, raw run clutter, and pre-formal probe result dumps are not kept as top-level documentation. The repo should preserve final theory, protocols, curated result audits, and reproducibility entrypoints.
